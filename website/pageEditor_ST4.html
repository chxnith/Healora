Profile Page

Technical Description

Semantic HTML

  1. Used semantic HTML elements like <header>, <nav>, and <footer>for structure, maintained proper heading hierarchy with <h2> and <h3>, and organized grouped items using <ul> and <li>.
  2. Screen Reader Support : Used alt text for images and combined emojis with text labels for clarity.
  3. Keyboard Navigation : Ensured keyboard accessibility and used proper <label>associations for form inputs.
  4. Visual Clarity : Used good color contrast, clear section headers, and progress bars with visual and textual indicators.
  5. Accessible Forms :Included labels, appropriate required attributes, and correct input types for all form fields.
  6. Responsive & Consistent : Responsive design ensures compatibility across screen sizes, with consistent navigation throughout the site.
  7. Date Picker- Uses native <input type="date" > for accessibility

This ensures usability for screen reader users, keyboard navigation, and those with visual or cognitive disabilities..

This profile page uses a combination of semantic HTML and modern CSS techniques to create a responsive, visually appealing layout.


HTML Structure:

  1. Semantic Layout:The page has a header with navigation, a main section with card content, and a footer with copyright and a link.
  2. Card-Based Design: Multiple <div class="card"> sections organize content into modular containers, including the profile card, bio section, goals, and contact form.
  3. Form Elements:A <form> uses properly associated <label> elements with input types like <date>, <radio>, and <text>, each with appropriate attributes.

CSS Techniques:

  1. 1. Flexbox and Grid:The layout combines Flexbox and Grid, with .container using display: flex and flex-wrap for responsiveness, .main-content using grid-template-columns for a 2-column layout, and .goals-grid implementing a 5-column grid for UN goals.
  2. Responsive Design:
    • Media queries at 1024px, 768px, and 480px breakpoints
    • Mobile-first approach with progressive enhancement
    • flex-direction: column` adjustment on smaller screens
  3. 3.Visual Styling:CSS uses variables for consistent colors, 'box-shadow' and 'border-radius' for card styling, and 'transition' effects for interactive hover states.
  4. 4.Skill Bars: Skill progress bars are built with nested <div> elements, where each bar’s width is controlled by a CSS class like .communication { width: 85%; }.
  5. 5.Form Styling: Radio buttons are styled using the :checked pseudo-class, with consistent spacing achieved through rem-based padding and margins.

Key Design Choices

  1. Color Scheme: The design uses blue (rgb(3, 57, 138)) for primary elements, green (#0f9b0f) for accents and interactive elements, and white cards with subtle shadows for a clean appearance.
  2. Typography: Text is styled with relative sizing using em/rem for scalability and font-weight contrast (e.g., bold titles vs regular body) to establish visual hierarchy.
  3. Mobile Optimization: The responsive design features a stacked layout on small screens, tap-friendly sizing for form elements, and simplified grids with 2 or 1 column using mobile-friendly CSS.

The page combines functional form elements with an engaging visual presentation, using modern CSS to create a responsive profile that works across devices while maintaining visual consistency. The semantic HTML structure ensures accessibility and proper document outlining.

Accessibility

This profile page implements strong accessibility practices through semantic HTML and ARIA-friendly styling:

  1. Semantic structure:with clear headings (<h2>, <h3>) and logical document flow
  2. Form accessibility:
    • Proper <label> associations using for,id pairing
    • Input fields with explicit type (date, email, etc.) and required attributes
    • Radio buttons styled accessibly with visible focus states
  3. Color contrast: meets WCAG standards (dark blue/white, green/white)
  4. Keyboard navigability: ensured via focusable interactive elements (links, form controls)
  5. Responsive design: prevents content clipping on small screens
  6. Text alternatives: for icons (emoji paired with visible labels)
  7. ARIA implicit roles: from semantic tags (e.g. <nav>, <form>)

The CSS enhances accessibility by maintaining readable text sizes,sufficient spacing, and clear visual feedback for interactions without relying solely on color.

Sitemap

Technical Description

HTML Structure and Key Tags

      1.Semantic Structure:
      • <svg>: Used to create the interactive sitemap diagram with connecting lines between nodes.
    1. <g>: SVG group elements to organize related nodes and paths.
    2. <path>: SVG elements that draw the connecting lines between sitemap nodes.
        <rect> and <text>: SVG elements that create the boxes and labels for each node.
  1. Purposeful Grouping:The SVG diagram is divided into three main sections:
    1. Main navigation pages(directly linked from Home)
    2. Content pages(secondary pages linked from Home)
    3. Editor pages(tertiary pages linked from Home)

CSS Styling Techniques

  1. Layout and Positioning:
    • Flexbox/Grid: Used implicitly through the container divs for responsive layout
    • SVG viewBox: viewBox="0 0 1200 900 ensures the diagram scales properly.
    • Absolute positioning: For precise placement of SVG elements.
  2. 2.Visual Design Elements:Color-coded nodes: Different fill colors distinguish between page types:
    • White for Home (root node)
    • Blue for main navigation
    • Green for content pages
    • Black for external links
    • Connecting lines: Dashed and solid lines show relationships between pages
    • Legend: Explains the color coding system for better user understanding

Functional Justifications

    1.SVG Diagram:
      Chosen over traditional HTML/CSS for its ability to create complex connecting lines
      Scalable without loss of quality
      Programmatically generated paths ensure precise connections
    2.Color Coding:
      Provides immediate visual categorization of page types
      Matched with legend for clarity
      High contrast for accessibility
    3.Responsive Design:
      SVG viewBox allows scaling to different screen sizes
      Container divs adjust to content flow
      Text sizes adapt to maintain readability

The combination of SVG for the diagram and semantic HTML/CSS for the surrounding content creates an effective visual representation of the website's structure while maintaining accessibility and responsive design principles.

Accessibility

  1. 1.Visual Accessibility:
    • Color-coded nodes with explanatory legend
    • High contrast design
    • Scalable SVG maintains clarity when zoomed
  2. Structural Clarity:
    • Clear hierarchy (Home → Main nav → Content pages)
    • Consistent node design with uniform information layout
    • Connecting lines show relationships
  3. Technical Accessibility:
    • Text remains selectable/readable in SVG
    • Responsive design works at any size
    • Logical left-to-right, top-to-bottom flow
  4. Cognitive Support:
    • Multiple visual cues (color + position + text)
    • Progressive disclosure of information
    • External links visually distinct

This creates an accessible experience for screen reader users, those with low vision, and users with cognitive differences through clear visual communication and logical structure.

Content Page

Technical Description

This page uses semantic HTML tags like <#header>, <#h1>, and <#nav> to organize content clearly and improve accessibility. The main content is wrapped in a .card-container div to create a focused, card-style layout. CSS styles set a clean font, comfortable line spacing, and a soft background for easy reading. The container is centered with a max width and padding for balanced spacing. Box shadows and rounded corners give a subtle 3D effect to the content area. Images are made responsive with width: 100% to adapt to different screen sizes. Proper margins and padding keep the layout neat and readable These HTML and CSS choices ensure the page is user-friendly, visually appealing, and accessible across devices.

Accessibility

In this content page, accessibility was carefully considered to ensure that all users, including those with disabilities, can access and understand the information. Meaningful alternative text was added to all images to support screen readers. The page structure uses clear and logical headings (<#h1>, <#h2>, <#h3>) to help users navigate the content easily. Navigation links are descriptive and organized for keyboard users, allowing easy tabbing through the menu and sidebar. The color contrast between text and background is sufficient for readability, and the font size is clear and legible. Additionally, the layout is responsive so that content remains accessible and easy to read on different screen sizes, supporting a wide range of devices and users. Overall, these efforts enhance usability and inclusivity on the page.

Challenges and Lessons Learned

While developing this web page, I faced numerous challenges, particularly related to layout alignment and unexpected bugs that affected the visual presentation and functionality across different screen sizes. These issues often disrupted the overall structure of the page, requiring extensive debugging and adjustments. To resolve them, I had to conduct a significant amount of research, experiment with different techniques, and carefully analyze the code. This process helped me deepen my understanding of responsive web design, especially in terms of using CSS properties such as flexbox, positioning, and media queries effectively.
