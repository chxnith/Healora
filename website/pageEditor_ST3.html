Feedback Page

Technical Description

The combined HTML and CSS files create a well-structured and visually pleasing feedback form page with a health and wellbeing theme. The HTML file utilizes semantic tags such as h1, h2, form, fieldset, and label to organize content and improve accessibility. Navigation is enabled using anchor tags within a nav element, with pages such as Home, Volunteer, and Feedback to help users navigate between. The form receives user input through input, textarea, and select fields, with form submission and reset buttons. The CSS is divided into two files: style.css targets the feedback form, using properties such as max-width, margin, padding, and box-shadow to center the form and create a clean, card-like appearance. Inputs and textareas are usability-friendly with even spacing and rounded edges. The rating section uses emojis and inline labels for an intuitive interface. Global.css, meanwhile, deals with the site-wide layout and branding, styling the top navigation bar with flexbox, background images, and drop shadows for a modern touch. Navigation links are styled with hover effects, and the sidebar utilizes fixed positioning and column layout to get the sticky vertical menu. The footer is styled with a dark background and centered text to provide a consistent closing page. Overall, the design choices prioritize clarity, accessibility, and responsiveness to ensure a smooth user experience across devices.

Accessibility

Accessibility renders sites accessible to all, including people with disabilities. Accessibility within this feedback form is possible using semantic HTML tags like form, label, and fieldset, which enable the content to be read by screen readers. Labels are properly linked to input fields, thereby facilitating easier navigation for assistive technology users. Clean text, high contrast, and well-spaced components also enhance the readability and interactivity of the form to all.

Team Page

Technical Description

Team page employs semantic HTML structure with CSS styling to neatly present the team members in a card-based layout. HTML elements like h1 and h2 are used for headings, and div elements enclose each member and their task. Anchors are utilized for navigation, and it is simple to navigate from one page to another. The team-style.css file utilizes flexbox to responsively align the team cards along the page. Each card holds an image and a hidden section with more information which is smoothly revealed on hover thanks to CSS transitions. The design utilizes consistent spacing, shadows, and rounded corners to visually balance the layout. The Global.css file contributes a site-wide consistent look and feel. It styles the top navigation bar with a background image and drop shadows and utilizes flexbox for positioning. The sidebar is fixed for convenient access, and the footer is styled with center-aligned text and contrasting colors. Altogether, all these practices constitute a clean, responsive, and easy-to-use team page.

Accessibility

Team page accessibility is met by using semantic HTML elements like header and h1 to enable the screen reader to read the hierarchy of the page. There is proper text contrast for reading and the layout is responsive for supporting varying screen widths. Smooth hover effects without the use of JavaScript are present, making the page accessible for assistive technology and keyboard users.

Content Page

Technical Description

The Nutrition & Healthy Eating page uses semantic HTML and clean CSS to organize health content in a readable, logical way. HTML headings like h1, h2, and p separate content into readable sections, and anchor tags enable navigation within the page and to other destinations on the site. Styling is accomplished using two CSS files. content_template.css styles the content section mainly using a centered card layout with padding, white background, and subtle shadows to produce a minimalist look. content_Global.css provides site-wide consistent styling, including fixed top navbar with background image, sidebar with quick section navigation, and styled footer. Flexbox, fixed positioning, and hover effects are used to produce improved layout and interaction. The approaches make the page look beautiful, user-friendly, and responsive on all devices.

Accessibility

The usability of the Nutrition & Healthy Eating webpage is made better by the use of a number of basic practices. The use of clearly defined and purposeful HTML tags, including h1, h2, and p, sets up the organization and structure of the content and thereby makes it easier to understand and navigate using screen reading software. The design also improves visual accessibility by using high-contrast text on light backgrounds and readable font sizing. The layout is flexible, ensuring information is available on different screen sizes and devices. Navigation links are clearly designated and can be easily accessed using a keyboard, thereby making it easy for users who rely on assistive technologies. Such choices increase the web page's inclusiveness and usability for a diverse set of users.

Challenges and Lessons Learned

While working on this website, I faced a few challenges that really helped me grow. One of the main struggles was making sure the layout worked well on all screen sizes. At first, elements like headers and buttons didn’t line up properly on smaller devices. I had to experiment with Flexbox and keep testing until it looked right. Another issue I came across was linking files correctly. Sometimes images or styles wouldn’t load because the path was wrong. That’s when I learned how important it is to understand relative paths and use ../ to navigate directories properly. Building forms—especially the feedback section—was also a learning moment. I wanted it to be user-friendly but also make sure people enter valid details. I ended up using attributes like required, pattern, and title to improve the user experience and keep the form functional. When it came time to combine my work with the rest of the team, there were a few clashes in styles and layout. I realized the importance of using a shared CSS file and sticking to consistent class names to avoid confusion. Overall, this project taught me a lot about writing clean, organized code and working as part of a team. It also showed me how valuable patience, testing, and problem-solving are when building a real website.

Compliance

While building the website, I followed basic JANET web publishing guidelines—mainly focusing on accessibility, appropriate content, and ethical design. Semantic HTML tags (e.g., header, main, and form) were used to enable users and screen readers to easily navigate the site. Every form control was supplied with clear labels, and even emoji ratings and other images had instructive alt text. The navigational scheme was kept simple and consistent across pages with a deliberate choice to exclude pop-ups or autoplaying media that might hinder functionality. I made certain that all material was original, appropriate, and free of copyright issues. While I didn't include more complex accessibility features like ARIA labels or a privacy policy—since the site doesn't collect any real information—I recognize that including these features would be essential for a properly functioning website. Overall, I hope that I have met JANET's basic standards while learning about how to design a user-friendly and ethically sound website.
