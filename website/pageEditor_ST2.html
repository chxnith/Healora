Home Page

Technical Description

I have designed and structured this webpage using HTML to create a clear and functional layout, and applied CSS to style it for a visually appealing and responsive experience. I used the <#header> tag to build the top navigation bar, which includes a logo, a heading, and navigation links placed within a <#nav> element to help users easily navigate to different pages. I organized the main content using a <#div class="card-container"> where I created multiple <#div class="card"> sections, each displaying an image, heading, short paragraph, and a "Learn More" link to present important health topics in an engaging way. I included a <#div class="right-section"> using flexbox to align a paragraph and an image side by side, which improves readability and layout balance. At the bottom, I added a panel and a <#footer> that contain quick links and validation links arranged using a table for better structure. For styling, I used .topbar with a background image, flexbox, and drop shadows to create a strong visual header, and styled the cards with box-shadow, border-radius, and hover effects to make them interactive. I also used CSS Grid in .card-container to evenly space the cards in four columns. Buttons such as .content-button and .redirect-button were styled with transitions and color changes on hover to improve usability. The .bottom-panel was designed with background color, padding, and positioned elements like the logo and table to maintain visual harmony. I applied global styles for consistency across pages, such as button styling and navigation link formatting, and added a styled footer to complete the page with proper branding and information.

Accessibility

In my homepage design, I focused on accessibility by adding `alt` text to all the images so that screen readers can describe them to visually impaired users. I also set the `lang="en"` attribute in the HTML tag to let browsers and assistive technologies know the page language. I used clear, readable fonts and made sure there was good color contrast between text and backgrounds to help users with visual difficulties. I organized the content with proper HTML structure like headings, nav, sections, and footer, making it easier to navigate for both users and screen readers. All buttons and links are clearly labeled so users can understand their purpose, and I used responsive layouts to make sure the content adjusts on all screen sizes, helping users on mobile and tablets.

Table Page

Technical Description

In this table page, I used standard HTML tags like <#table>, <#thead>, <#tbody>, <#tr>, <#th>, and <#td> to create a clear and organized structure for displaying disease-related information. The <#thead> is used to separate the headings from the data rows inside <#tbody>, making the content easier to read and understand. I applied CSS to center the table with margin: auto, added padding for spacing, and used border-collapse: collapse for a clean layout. To enhance design and usability, I used box-shadow to give the table a modern card-like effect and added a hover effect on table rows using tbody tr:hover, which highlights the row when the user hovers over it. The header row is styled with a blue background using background-color: #2d89ef to visually separate it from the data. These HTML and CSS choices make the table look neat, readable, and user-friendly while helping users quickly scan important health-related details.

Accessibility

In this table page, I focused on accessibility by using semantic HTML elements such as <#table>, <#thead>, and <#tbody>, which help screen readers understand the structure of the content. The use of clear headings inside <#th> tags provides meaningful labels for each column, making it easier for visually impaired users to navigate the table. I also used alt text for the logo image to describe its content for screen readers. Additionally, the navigation bar is structured with simple <#a> tags and clear text, which improves keyboard navigation. By setting the lang="en" attribute in the HTML tag, I ensured that assistive technologies recognize the page language correctly. These techniques make the page more user-friendly and accessible to everyone, including users with disabilities.

Content Page

Technical Description

This content page uses semantic HTML and well structured CSS to create an informative and user friendly layout. Key HTML tags like <#header>, <#nav>, <#section>, and <#footer> ensure the content is organized for both users and assistive technologies. Headings (<#h1>, <#h2>) and IDs allow easy navigation, supported by a sidebar with anchor links for quick access to sections. The CSS separates layout and design, with one file (content_Global.css) handling global styling like the fixed top navigation and sidebar, and another (content_template.css) focusing on the main content area styling such as card layout, background, and typography. Flexbox and fixed positioning ensure a responsive and accessible interface, while shadows, padding, and hover effects enhance readability and visual appeal. The decision to split the CSS was to avoid style conflicts—especially color issues—between the main site and content pages, ensuring clarity and consistency.

Accessibility

In this content page, accessibility has been addressed by using semantic HTML elements such as <#header>, <#nav>, <#main>, and proper heading tags (<#h1>, <#h2>, etc.) to create a logical and easy to navigate structure for screen readers. All images include descriptive alt text, helping visually impaired users understand the content. Anchor links in the sidebar and the "back to top" arrow allow for smooth keyboard navigation and quick access to key sections. The use of clear fonts, good contrast, and responsive design also improves readability for all users, ensuring the page is user-friendly and accessible across devices.

Challenges and Lessons Learned

When I was creating this web page, I faced many alignment issues and bugs, especially when working with different screen sizes and layout structures. This required me to do a lot of research, trial and error, and debugging to fix problems and ensure the page displayed correctly on all devices. These challenges pushed me to work hard and develop my problem-solving skills. Through this process, I learned a lot about HTML tags, advanced CSS styling techniques like flexbox and positioning, as well as the importance of responsive design and cross browser compatibility. I also became more familiar with organizing CSS files efficiently by separating global and page-specific styles. Overall, it was a challenging but rewarding learning experience that helped me grow as a web developer.
